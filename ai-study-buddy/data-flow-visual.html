  1. Press Ctrl+P (or Cmd+P on Mac) to open the print dialog.
  2. Change the "Destination" to "Save as PDF".
  3. Click "Save".

1. AI Chat & RAG Data Flow

graph TD A[User types message or uploads document] --> B{ChatInterface Component}; B --> C[Update React State: messages & studyContext]; C --> D[Call geminiService.getChatResponseStream]; D --> E[Construct Prompt with RAG Context]; E --> F[API Call to Google Gemini]; F -- Stream Chunks --> D; D -- Yield Chunks --> B; B --> G[Live-update UI with streaming text]; C --> H[Save updated chat to LocalStorage];

Explanation:

  • 1. User Input: The user interacts with the `ChatInterface`, either by typing a message or uploading a `.txt`/`.pdf` file to provide context for Retrieval-Augmented Generation (RAG).
  • 2. State Update: The React component immediately updates its local state. The new message is added to the conversation history, and the content of any uploaded file is parsed and stored in the `studyContext` state.
  • 3. Service Call: The component calls the `getChatResponseStream` function located in `geminiService.ts`, passing the complete chat history and the `studyContext`.
  • 4. API Request: The service constructs a request for the Google Gemini API. If RAG context is present, a specialized system prompt is included that instructs the model to prioritize answers from the user's document.
  • 5. Streaming Response: The Gemini API processes the request and streams the response back chunk by chunk, which is consumed by the service function.
  • 6. Live UI Update: As each text chunk arrives, it is yielded to the `ChatInterface` component, which updates its state. This results in the AI's response appearing on the screen in real-time.
  • 7. Persistence: A `useEffect` hook saves the entire updated conversation history to the browser's `LocalStorage`, ensuring the chat is preserved across sessions.

2. Structured Data Generation (Quiz / Planner)

graph TD A[User enters topic and clicks "Generate"] --> B{UI Component (e.g., QuizGenerator)}; B --> C[Set isLoading state to true]; C --> D[Call geminiService function (e.g., generateQuiz)]; D --> E[API Call to Gemini with JSON Schema]; E -- Returns Validated JSON --> D; D -- Returns Parsed Object --> B; B --> F[Set isLoading to false, update state with data]; F --> G[Render Quiz/Plan to user];

Explanation:

  • 1. User Input: The user provides a topic for a quiz or a goal for a study plan in the corresponding UI component.
  • 2. Loading State: The component sets its `isLoading` state to `true`, which triggers the display of a loading indicator, providing immediate feedback to the user.
  • 3. Service Call: The appropriate function in `geminiService.ts` is called (e.g., `generateQuiz`).
  • 4. Structured API Request: The service makes a request to the Google Gemini API. Crucially, this request includes `responseMimeType: "application/json"` and a `responseSchema` that defines the exact structure of the expected JSON output.
  • 5. JSON Response: The Gemini model generates content that strictly adheres to the provided schema and returns it as a JSON string.
  • 6. Parsing & State Update: The service parses the JSON string into a JavaScript object. The UI component receives this object, updates its state with the new data, and sets `isLoading` to `false`.
  • 7. UI Render: React re-renders the component, displaying the fully interactive quiz or the structured study plan to the user.

3. Client-Side Gamification & Tracking

graph TD A[User completes an action
e.g., finishes quiz] --> B{UI Component}; B --> C[Call tracking utility function
e.g., logQuizCompletion]; C --> D[Read current stats from LocalStorage]; D --> E[Update stats object & check streak]; E --> F[Write updated stats to LocalStorage]; G[Dashboard Component] -- On Load --> H[Read stats from LocalStorage]; H --> I[Render stats, streak, and badges];

Explanation:

  • 1. User Action: A user performs a trackable action, such as submitting a quiz, reviewing a flashcard, or checking off a task in the study planner.
  • 2. Utility Call: The relevant UI component calls a function from `utils/tracking.ts` (e.g., `logQuizCompletion`).
  • 3. Read: The tracking utility reads the current `DashboardStats` object from `LocalStorage`.
  • 4. Update: The utility updates the statistics in the object (e.g., increments `quizzesCompleted`). It also checks the `lastActivityDate` to determine if the daily study streak should be updated.
  • 5. Write: The updated `DashboardStats` object is immediately written back to `LocalStorage`, persisting the user's progress.
  • 6. Dashboard Load: When the `Dashboard` component loads (or when the user navigates to it), it reads the latest stats directly from `LocalStorage` to ensure the most up-to-date information is always displayed.
